1. Adding the Calculated Field:

We can't directly modify a view in SQLite. However, we can create a new view that inherits the data from the existing one and adds the calculated field for the time difference.

Here's the modified SQL code:

sql CREATE VIEW VW_PRODUCT_V2 AS SELECT P.ID_PROD AS PRODUCT_ID, C.DS_CATEG AS CATEGORY, B.DS_BRAND AS BRAND, M.DS_MODEL AS MODEL, P.DS_PROD AS PRODUCT_NAME, A.VL_OF AS VL_OF, A.VL_PER AS VL_PER, CASE WHEN A.VL_OF > 0 THEN ROUND((A.VL_PER - A.VL_OF) / A.VL_OF * 100, 2) ELSE 0.00 END AS DISCOUNT, P.URL_IMG AS PRODUCT_IMAGE, A.URL_AFFILIATE AS AFFILIATE_LINK, P.REG_DH_INS AS REG_DH_INS, /* New calculated field */ CAST(julianday(CURRENT_TIMESTAMP) - julianday(P.REG_DH_INS) * 24 * 60 AS INTEGER) AS TIME_DIFF_MINUTES FROM TB_PROD P INNER JOIN TB_CATEG C ON P.ID_CATEG = C.ID_CATEG INNER JOIN TB_BRAND B ON P.ID_BRAND = B.ID_BRAND LEFT JOIN TB_MODEL M ON P.ID_MODEL = M.ID_MODEL LEFT JOIN TB_PROD_AFFILIATE PA ON P.ID_PROD = PA.ID_PROD LEFT JOIN TB_AFFILIATE A ON PA.ID_AFFILIATE = A.ID_AFFILIATE;

Explanation:

  • CURRENT_TIMESTAMP: This function retrieves the current date and time in SQLite.
  • julianday(date_expression): This function converts a date or datetime expression to a Julian Day number.
  • * 24 * 60: This part converts the difference in Julian days to minutes.
  • CAST(... AS INTEGER): This ensures the result is stored as an integer representing the time difference in minutes.
