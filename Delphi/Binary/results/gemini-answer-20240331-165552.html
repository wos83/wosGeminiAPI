1. Utilize Virtual Columns:

Since you're working with a view (VW_PRODUCT_V1), directly adding a new column for the time difference isn't possible. However, SQLite offers virtual columns that allow calculations on the fly during data retrieval.

Here's the modified view definition:

sql CREATE VIEW VW_PRODUCT_V1 AS SELECT P.ID_PROD AS PRODUCT_ID, C.DS_CATEG AS CATEGORY, B.DS_BRAND AS BRAND, M.DS_MODEL AS MODEL, P.DS_PROD AS PRODUCT_NAME, A.VL_OF AS VL_OF, A.VL_PER AS VL_PER, CASE WHEN A.VL_OF > 0 THEN ROUND((A.VL_PER - A.VL_OF) / A.VL_OF * 100, 2) ELSE 0.00 END AS DISCOUNT, P.URL_IMG AS PRODUCT_IMAGE, A.URL_AFFILIATE AS AFFILIATE_LINK, P.REG_DH_INS AS REG_DH_INS, /* New virtual column for time difference in seconds */ CAST(strftime('%s', 'now') - strftime('%s', REG_DH_INS) AS INTEGER) AS TIME_DIFF_SEC FROM TB_PROD P INNER JOIN TB_CATEG C ON P.ID_CATEG = C.ID_CATEG INNER JOIN TB_BRAND B ON P.ID_BRAND = B.ID_BRAND LEFT JOIN TB_MODEL M ON P.ID_MODEL = M.ID_MODEL LEFT JOIN TB_PROD_AFFILIATE PA ON P.ID_PROD = PA.ID_PROD LEFT JOIN TB_AFFILIATE A ON PA.ID_AFFILIATE = A.ID_AFFILIATE;

Explanation:

  • We've added a new virtual column named TIME_DIFF_SEC.
  • CAST converts the result to an integer.
  • strftime('%s', 'now') gets the current timestamp in Unix epoch format (seconds since 1970-01-01 00:00:00 UTC).
  • strftime('%s', REG_DH_INS) gets the REG_DH_INS timestamp in Unix epoch format.
  • Subtracting these two values gives the time difference in seconds.

2. Accessing the Data:

Now, when you query the VW_PRODUCT_V1 view, you'll have access to the TIME_DIFF_SEC column. For example:

sql SELECT * FROM VW_PRODUCT_V1 WHERE PRODUCT_ID = 123;

This will return all information about product ID 123, including the newly calculated TIME_DIFF_SEC representing the time difference in seconds since the product was inserted.

Additional Notes:

  • This approach avoids modifying the original table structure.
  • Remember that virtual columns are calculated during each query and aren't stored persistently.

This solution provides a way to calculate the desired time difference within the view itself. If you need persistent storage of the time difference, you might consider creating a trigger or modifying the data insertion process to calculate and store the difference in a separate column within the TB_PROD table.
