Como usar o SHRINK via linha de comando:

1. Abrindo o Prompt de Comando:

  • Pressione Windows + R para abrir a caixa de diálogo Executar.
  • Digite cmd e pressione Enter.

2. Conectando-se ao SQL Server:

  • Digite o seguinte comando:

sql sqlcmd -S <nome_do_servidor> -U <nome_do_usuario> -P <senha>

  • Substitua <nome_do_servidor> pelo nome do seu servidor SQL Server.
  • Substitua <nome_do_usuario> pelo nome de usuário do banco de dados.
  • Substitua <senha> pela senha do usuário do banco de dados.

3. Reduzindo o tamanho do banco de dados:

  • Digite o seguinte comando:

```sql USE GO

DBCC SHRINKDATABASE (, ) GO ```

  • Substitua <nome_do_banco_de_dados> pelo nome do banco de dados que você deseja reduzir.
  • Substitua <tamanho_desejado> pelo novo tamanho desejado para o banco de dados (em MB).

4. Opções adicionais:

  • minimum: Especifica o tamanho mínimo que o banco de dados pode ter após a redução.
  • target: Especifica o tamanho ideal que o banco de dados deve ter após a redução.

5. Exemplos:

  • Reduzir o banco de dados AdventureWorks para 10 GB:

```sql USE AdventureWorks GO

DBCC SHRINKDATABASE (AdventureWorks, 10000) GO ```

  • Reduzir o banco de dados Northwind para 5 GB com um tamanho mínimo de 2 GB:

```sql USE Northwind GO

DBCC SHRINKDATABASE (Northwind, 5000, 2000) GO ```

6. Considerações:

  • Antes de reduzir o tamanho do banco de dados, faça um backup completo.
  • Certifique-se de que o novo tamanho seja suficiente para armazenar todos os dados do banco de dados.
  • A redução do tamanho do banco de dados pode levar algum tempo.

7. Recursos adicionais:

  • Microsoft Docs: SHRINKDATABASE (Transact-SQL): [URL inválido removido]
  • Reduzir um volume básico (Windows Server): https://docs.microsoft.com/pt-br/windows-server/storage/disk-management/shrink-a-basic-volume

8. Dica:

Se você não tiver certeza do tamanho desejado para o banco de dados, pode usar o comando DBCC SHRINKDATABASE sem especificar um tamanho. Isso reduzirá o banco de dados para o menor tamanho possível.

9. Observação:

Este comando só pode ser usado em bancos de dados que usam o modelo de recuperação completa ou bulk-logged.
