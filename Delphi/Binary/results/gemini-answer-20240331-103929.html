Configurando o Delphi 10.2.3 para depuração de um aplicativo IntraWeb no IIS:

1. Definindo o projeto como CGI:

  • Abra o projeto IntraWeb no Delphi 10.2.3.
  • No menu principal, selecione Project > Options.
  • Na guia Application, altere o tipo de projeto para CGI Application.
  • Especifique o nome do arquivo CGI principal (geralmente main.exe).

2. Configurando o IIS:

  • Abra o Gerenciador de Serviços de Informações da Internet (IIS).
  • Navegue até o site que hospeda o aplicativo IntraWeb.
  • Clique com o botão direito do mouse no nome do site e selecione Adicionar Aplicativo Virtual.
  • No campo Nome do alias, digite o nome do alias do aplicativo (por exemplo, intraweb).
  • No campo Caminho físico, especifique o diretório onde o arquivo main.exe está localizado.
  • Selecione a opção Permitir scripts.
  • Clique em OK para criar o aplicativo virtual.

3. Habilitando a depuração no IIS:

  • Abra o Painel de Controle do Windows.
  • Selecione Programas e Recursos > Ativar ou desativar recursos do Windows.
  • Expanda Internet Information Services (IIS) > Serviços de Desenvolvimento de Aplicativos.
  • Marque a caixa de seleção Depuração de ASP.NET.
  • Clique em OK para instalar o recurso de depuração.

4. Configurando o Delphi para depuração:

  • No Delphi, acesse o menu Run > Debug Options.
  • Na guia Debugger, selecione IIS como o tipo de depurador.
  • Especifique a porta que o IIS usará para depuração (geralmente 8080).
  • Marque a caixa de seleção Habilitar depuração remota.
  • Clique em OK para salvar as configurações de depuração.

5. Iniciando a depuração:

  • Pressione F9 no Delphi para iniciar a depuração.
  • O Delphi se conectará ao IIS e iniciará o processo de depuração.
  • Você poderá usar os recursos de depuração do Delphi para examinar o código, definir pontos de interrupção e avaliar variáveis.

Dicas adicionais:

  • Certifique-se de que o IIS esteja configurado para usar o mesmo pool de aplicativos que o Delphi.
  • Se você estiver usando o IIS Express, configure o Delphi para usar a porta 8080.
  • Você pode usar o log do IIS para verificar se há erros durante a depuração.
